# On-chain Data Q&A Bot / 链上数据问答 Bot

> 用户用自然语言提问，AI 查询链上数据并回答

## 解决什么问题

普通用户查询链上数据需要理解 Sui RPC 接口、编写查询脚本或使用区块浏览器的高级功能，门槛极高。"过去 24 小时 Sui 上交易量最大的协议是哪个？"这类简单问题无法快速获得答案。本项目提供自然语言问答界面，用户用日常语言提问，AI Agent 自动将问题转换为链上数据查询，聚合结果后用自然语言回答，大幅降低链上数据获取门槛。

## 核心功能

- 自然语言问答：支持任意关于 Sui 链上数据的问题
- 自动查询生成：AI 将自然语言转换为 Sui RPC 查询和索引器 API 调用
- 多数据源聚合：链上 RPC、索引服务、协议子图等统一查询
- 回答带来源引用：每个回答附带查询路径和原始数据链接，可验证

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| 链上数据 | 直接查询 Sui RPC 获取交易、对象、事件等原始数据 |
| 事件（Event） | 查询和分析协议发出的事件数据 |
| Move 对象模型 | 读取协议共享对象获取状态数据 |
| 动态字段 | 查询对象的动态字段获取附加数据 |

## 实现方案

### 智能合约 (Move)
- **`qa_record` 模块**：定义 `QARecord` 对象，字段包括 `question_hash: vector<u8>`、`answer_hash: vector<u8>`、`data_sources: vector<String>`、`confidence: u64`、`timestamp: u64`
- **`query_cache` 模块**：`cache_result(question_hash: vector<u8>, result_bytes: vector<u8>, ttl_epochs: u64)` 缓存常见查询结果减少重复 RPC 调用
- **`data_anchor` 模块**：`anchor_data(source: String, data_hash: vector<u8>)` 为回答中的关键数据提供链上锚定确保可信
- 关键数据结构：`QueryResult { query_type: u8, result_json: vector<u8>, block_height: u64, fetch_timestamp: u64 }`

### 前端 / 后端
- **前端**：React + Next.js，聊天式问答界面、数据可视化卡片、历史问答记录
- **后端**：Python Agent 服务，问题理解、查询规划、数据聚合、回答生成
- 关键功能：对话框、数据图表嵌入、查询收藏

### AI / Agent 部分
- 问题理解：GPT-4o 将自然语言问题解析为结构化查询意图（目标数据、时间范围、聚合方式）
- 查询规划：ReAct Agent 根据意图选择最优数据源和查询路径（RPC / 索引器 / 协议子图）
- 数据聚合：执行多步查询，合并结果
- 回答生成：GPT-4o 将聚合数据转化为自然语言回答，附带数据来源和图表
- Agent 工作流：接收问题 → 意图解析 → 查询规划 → 多源数据获取 → 聚合分析 → 自然语言回答

## 技术架构

用户在前端聊天框输入自然语言问题。GPT-4o 解析问题意图并生成查询计划（需要哪些数据源、查询参数、聚合逻辑）。ReAct Agent 按计划执行多步查询：先查 Sui RPC 获取基础数据，再查索引服务获取聚合统计，必要时调用协议特定 API。结果数据经清洗聚合后，GPT-4o 生成自然语言回答并嵌入可视化图表。关键查询结果通过 QARecord 对象链上锚定，确保数据可追溯和可验证。

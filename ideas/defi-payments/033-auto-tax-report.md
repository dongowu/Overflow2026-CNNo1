# 自动税务报告 / Auto Tax Report

> 追踪所有链上交易，自动计算资本利得，生成税务报告

## 解决什么问题

加密货币投资者在报税时需要追踪所有链上交易并计算资本利得，过程繁琐且容易出错。不同司法管辖区的税法各异（FIFO/LIFO/加权平均），手动处理几乎不可能。多个链和钱包的交易记录分散，难以汇总。自动税务报告工具索引用户的所有链上交易，按适用税法自动计算资本利得，生成可直接用于报税的文档。

## 核心功能

- 全交易追踪：自动索引用户在 Sui 上的所有交易（转账、兑换、DeFi 交互）
- 多会计方法：支持 FIFO、LIFO、加权平均等成本计算方法
- 资本利得计算：自动区分短期和长期资本利得，按适用税率计算
- 税务报告导出：生成 CSV/PDF 格式的税务报告，包含汇总和明细

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | TaxReport 作为独立 Object，存储计算结果和报告元数据 |
| PTB | 原子化执行"拉取交易 → 计算利得 → 生成报告" |
| 动态字段 | 存储交易明细、成本基础和利得计算结果 |
| 事件 (Events) | 订阅链上交易事件构建用户交易历史 |
| 预言机集成 | 获取历史价格数据用于成本基础计算 |

## 实现方案

### 智能合约 (Move)
- `tax_registry` 模块：注册用户钱包地址和适用的税务管辖区
- `cost_basis` 模块：按选定会计方法计算成本基础
- `gain_calculator` 模块：计算每笔交易产生的资本利得或损失
- `report_generator` 模块：生成链上可验证的税务报告摘要
- 关键数据结构：`TaxReport { owner: address, tax_year: u16, jurisdiction: String, total_gains: u64, total_losses: u64, transactions: vector<TaxEvent>, method: u8 }`

### 前端 / 后端
- 技术栈：Next.js + Sui TypeScript SDK + PDF 生成库
- 关键页面：Dashboard（税务总览）、Transaction Log（交易日志）、Capital Gains（资本利得明细）、Export Report（导出报告）
- 后端索引链上事件并执行税务计算

## 技术架构

用户连接钱包后，后端索引器扫描其所有链上交易记录，通过预言机获取每笔交易时的历史价格。cost_basis 模块按用户选择的会计方法（FIFO/LIFO/加权平均）计算每笔资产的成本基础。gain_calculator 为每笔卖出交易计算资本利得或损失。所有计算结果通过 PTB 存储到链上 TaxReport 对象中，确保数据可验证。前端提供可视化报告和 PDF/CSV 导出功能。

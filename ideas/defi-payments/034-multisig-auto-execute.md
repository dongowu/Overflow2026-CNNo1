# 多签 + 自动执行 / Multisig + Auto Execute

> 多签钱包结合自动化规则，满足条件时自动执行 PTB，无需人工签名

## 解决什么问题

传统多签钱包需要指定数量的签名者手动确认每笔交易，对于定期支付（工资、供应商付款）和条件触发交易（再平衡、止损），每次都需人工签名非常低效。多签 + 自动执行将多签的安全性与自动化的便利性结合，首次多签批准策略后，满足条件的交易自动执行，无需重复签名。

## 核心功能

- 策略多签审批：新增自动执行策略需要多签者共同批准
- 条件自动执行：策略定义触发条件（时间、价格、事件），满足后自动执行
- 限额分层：小额自动执行、中额单签、大额全签，灵活平衡安全与效率
- 策略审计日志：所有自动执行记录在链上，多签者可随时审查和撤销策略

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | MultisigVault 作为共享对象，管理资金和策略 |
| PTB | 原子化执行"条件检查 → 策略验证 → 资金转移 → 日志记录" |
| 多签 (MultiSig) | 策略创建和修改需要多签确认 |
| 动态字段 | 存储签名者列表、策略列表和执行历史 |
| 共享对象 | 金库作为共享对象，支持 Keeper 触发自动执行 |

## 实现方案

### 智能合约 (Move)
- `multisig_vault` 模块：管理多签金库的资金存取
- `policy_engine` 模块：定义和管理自动执行策略（条件、限额、频率）
- `auto_executor` 模块：验证条件并执行自动转账
- `audit_log` 模块：记录所有自动执行的详细信息
- 关键数据结构：`AutoPolicy { policy_id: ID, condition: Condition, max_amount: u64, recipient: address, frequency: u64, approved_by: vector<address>, active: bool }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Vault Dashboard（金库总览）、Policy Management（策略管理）、Execution Log（执行日志）、Signers（签名者管理）
- 后端 Keeper 监控条件并触发自动执行

## 技术架构

多签者共同创建 MultisigVault 并存入资金。新增自动执行策略时需要达到多签阈值确认。策略定义了触发条件（如每月 1 号、价格低于某值）、执行动作（转账金额和目标）和限额。Keeper 服务持续检查条件，满足时通过 PTB 自动执行：验证策略活跃 → 检查限额 → 执行资金转移 → 记录到审计日志。任何多签者可发起撤销策略的提案，多签通过后立即停用。

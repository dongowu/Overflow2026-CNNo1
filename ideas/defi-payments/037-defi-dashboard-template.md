# DeFi 仪表盘模板 / DeFi Dashboard Template

> 开源的 Sui DeFi 数据仪表盘，项目方可快速定制部署

## 解决什么问题

每个 Sui DeFi 项目都需要一个数据仪表盘来展示 TVL、交易量、用户数等核心指标。但从零开发仪表盘需要大量前端工作和链上数据索引，项目方重复投入资源。DeFi 仪表盘模板提供一套开源、可定制的仪表盘框架，项目方只需配置合约地址和参数即可快速部署专业的 DeFi 数据展示平台。

## 核心功能

- 预置数据面板：TVL 趋势、24h 交易量、活跃用户、协议收入等核心指标
- 图表组件库：折线图、柱状图、饼图等丰富的可视化组件
- 一键配置部署：通过配置文件指定合约地址和指标定义，自动生成仪表盘
- 实时数据更新：通过 WebSocket 订阅链上事件，数据实时刷新

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| 事件 (Events) | 订阅协议事件构建实时数据面板 |
| 动态字段 | 读取链上对象的动态字段获取协议状态 |
| 共享对象 | 查询协议的共享对象获取 TVL 等数据 |
| Coin 标准 | 解析各类代币余额和交易量 |
| PTB | 构建批量查询交易获取多个协议数据 |

## 实现方案

### 智能合约 (Move)
- `data_feed` 模块：标准化协议数据输出接口，供仪表盘读取
- `metrics_collector` 模块：聚合计算 TVL、交易量等核心指标
- `event_standard` 模块：定义标准化的事件格式供索引器使用
- 关键数据结构：`ProtocolMetrics { tvl: u64, volume_24h: u64, users_24h: u32, fees_24h: u64, updated_at: u64 }`

### 前端 / 后端
- 技术栈：Next.js + Recharts + TailwindCSS + Sui TypeScript SDK
- 关键页面：Overview（总览）、TVL Chart（TVL 趋势）、Volume Chart（交易量趋势）、User Metrics（用户指标）、Config Panel（配置面板）
- 后端使用 Sui Event Indexer 聚合链上数据

## 技术架构

项目方 clone 模板仓库后，编辑配置文件指定自己的合约地址、代币类型和指标定义。模板自动通过 Sui TypeScript SDK 订阅合约事件和查询共享对象状态。data_feed 模块提供标准化的数据输出接口，仪表盘通过 PTB 批量查询多个协议指标。Recharts 组件自动渲染趋势图和实时数据。部署后即可获得专业的 DeFi 数据展示平台，支持进一步定制化调整样式和布局。

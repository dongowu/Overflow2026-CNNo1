# Move 金融合约模板库 / Move Finance Contract Templates

> 预构建的代币、金库、托管、支付合约模板，CLI 一键生成

## 解决什么问题

每个 DeFi 项目都需要实现相似的基础功能（代币创建、金库管理、支付处理、托管合约），开发者重复编写类似的 Move 合约代码。这不仅浪费时间，还容易引入安全漏洞。Move 金融合约模板库提供经过审计的预构建合约模板，开发者通过 CLI 命令一键生成定制化的 Move 合约代码，加速开发并提高安全性。

## 核心功能

- 合约模板库：涵盖代币、金库、托管、支付、NFT、治理等常见合约类型
- CLI 一键生成：`sui-finance create token --name MyToken --decimals 9` 生成完整合约
- 参数化定制：通过命令行参数和配置文件定制合约行为
- 安全审计保障：所有模板经过安全审计，附带测试套件和文档

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Coin 标准 | 代币模板使用 Sui Coin<T> 标准 |
| Move 对象模型 | 所有模板基于 Sui 对象模型构建 |
| PTB | 模板中的复合操作使用 PTB 模式 |
| 动态字段 | 金库和托管模板使用动态字段存储状态 |
| TransferPolicy | 支付模板使用 TransferPolicy 控制规则 |

## 实现方案

### 智能合约 (Move)
- `token_template` 模块：可定制的代币合约模板（名称、精度、供应量、铸造规则）
- `vault_template` 模块：资金金库模板（存取、份额代币、费用收取）
- `escrow_template` 模块：托管合约模板（条件锁定、释放、争议处理）
- `payment_template` 模块：支付处理模板（多币种、自动兑换、结算）
- 关键数据结构：每个模板包含完整的模块结构和配置参数

### 前端 / 后端
- 技术栈：Rust/TypeScript CLI + Handlebars 模板引擎
- 关键组件：CLI 工具（sui-finance）、模板仓库（Git）、代码生成器、测试框架
- 输出：可直接编译部署的 Move 合约代码 + 测试文件 + 部署脚本

## 技术架构

开发者安装 CLI 工具后，选择需要的合约类型并配置参数。CLI 通过 Handlebars 模板引擎将参数注入预构建的 Move 合约模板，生成完整的合约代码。生成的代码包含合约模块、单元测试和部署脚本。开发者可直接编译部署到 Sui 测试网，也可基于生成的代码进行二次开发。模板库通过 Git 管理，社区可贡献新模板和改进现有模板。

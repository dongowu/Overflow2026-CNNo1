# 链上事件索引框架 / On-Chain Event Indexing Framework

> 开发者友好的 Sui 事件索引 SDK，支持自定义过滤和实时推送

## 解决什么问题

Sui DeFi 开发者需要实时追踪链上事件来更新前端状态和触发后端逻辑。但现有的事件索引工具配置复杂、不支持自定义过滤、实时性差。开发者往往需要自己搭建索引服务，重复投入大量基础设施工作。链上事件索引框架提供开箱即用的事件索引 SDK，支持自定义过滤器、实时 WebSocket 推送和历史查询，让开发者专注于业务逻辑。

## 核心功能

- 声明式事件过滤：通过配置文件定义关注的事件类型和过滤条件
- 实时 WebSocket 推送：新事件产生时毫秒级推送到订阅者
- 历史事件查询：支持按时间范围、发送者、事件类型等维度查询历史事件
- 自动状态聚合：从事件流自动聚合出业务状态（如用户持仓、交易量统计）

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| 事件 (Events) | 框架的核心索引对象 |
| Move 对象模型 | 通过事件关联对象状态 |
| 动态字段 | 解析事件中引用的动态字段数据 |
| PTB | 构建批量事件查询交易 |
| 共享对象 | 查询共享对象状态补充事件数据 |

## 实现方案

### 智能合约 (Move)
- `event_standards` 模块：定义标准化的事件格式规范
- `event_emitter` 模块：提供辅助函数帮助协议发出标准化事件
- `state_checkpoint` 模块：定期在链上记录状态检查点，加速索引恢复
- 关键数据结构：`IndexedEvent { event_type: String, sender: address, timestamp: u64, data: vector<u8>, object_id: Option<ID> }`

### 前端 / 后端
- 技术栈：TypeScript SDK (NPM Package) + WebSocket Server + PostgreSQL/ClickHouse
- 关键组件：SDK 客户端、索引器引擎、过滤器引擎、状态聚合器、REST API
- 输出：NPM 包供开发者安装，Docker Compose 一键部署索引服务

## 技术架构

开发者安装 SDK 后，通过配置文件声明关注的事件（合约地址 + 事件类型 + 过滤条件）。索引器引擎通过 Sui Full Node 的 WebSocket 订阅实时事件，按过滤器筛选后推送到开发者注册的回调函数。历史事件从链上 checkpoint 同步并存储在本地数据库。状态聚合器从事件流中自动计算业务状态（如用户余额变化、协议 TVL 变化）。提供 REST API 和 GraphQL 接口供前端查询。

## 难度评估

| 维度 | 难度 | 说明 |
|------|------|------|
| Move 合约 | ⭐⭐ | 事件标准模块相对简单 |

| 集成复杂度 | ⭐⭐⭐⭐ | 高性能索引引擎需要处理大量事件流 |

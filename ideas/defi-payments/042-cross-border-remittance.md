# 跨境汇款协议 / Cross-Border Remittance Protocol

> 发送方用法币入金，链上路由最优路径，收款方在当地取出法币

## 解决什么问题

传统跨境汇款手续费高达 3-10%，到账时间 1-5 个工作日，过程不透明。依赖 SWIFT 网络和多个中间银行，每经手一层就扣减一次费用。跨境汇款协议利用 Sui 链上流动性池和 DEX 路由，为跨境转账提供最低成本和最快速度的路径。发送方法币入金后，系统自动选择最优链上路径完成兑换和转账，收款方在当地取出法币。

## 核心功能

- 法币入金/出金：集成法币通道，支持银行转账和移动支付入金
- 最优路径路由：自动在多个 DEX 和流动性池间选择汇率最优的兑换路径
- 实时汇率透明：展示实际汇率、手续费和到账金额，无隐藏费用
- 多国家/地区支持：支持主要货币对的跨境转账

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| PTB | 原子化执行"入金 → 多跳兑换 → 链上转账 → 出金准备" |
| Coin 标准 | 处理各种货币对应的链上代币 |
| DeepBook 集成 | 通过 DEX 实现最优汇率兑换 |
| 动态字段 | 存储汇率、手续费和转账路径 |
| 共享对象 | 流动性池作为共享对象供路由选择 |

## 实现方案

### 智能合约 (Move)
- `remittance_factory` 模块：创建跨境转账订单
- `route_optimizer` 模块：计算最优兑换路径（可能经过多个 DEX）
- `liquidity_pool` 模块：管理各货币对的流动性
- `settlement` 模块：处理出金请求和到账确认
- 关键数据结构：`RemittanceOrder { sender: address, recipient: address, send_amount: Coin<T>, receive_currency: Type, route: vector<RouteStep>, status: u8, created_at: u64 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + 法币通道 SDK
- 关键页面：Send Money（发起汇款）、Route Preview（路径预览和费用明细）、Track Transfer（追踪转账状态）、Recipient Management（收款人管理）
- 后端对接法币通道服务提供入金/出金能力

## 技术架构

发送方在前端发起汇款，选择发送金额和收款币种。route_optimizer 搜索所有可用 DEX 路径，计算最优汇率和最低手续费。发送方确认后通过 PTB 原子化执行：法币入金 → 按最优路径兑换 → 转入收款方地址的法币出金池。收款方收到通知后在本地通过出金通道提取法币。整个过程费用透明、路径可追踪，通常 10 分钟内完成（主要等待法币通道处理时间）。
